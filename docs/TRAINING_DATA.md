# Training Data Reference

ProtePilot maintains two distinct training datasets. They serve different
models and are generated by different pipelines. This document explains what
each file contains, how it is produced, and how to regenerate it.

---

## Dataset Summary

| File | Rows | Model | Script |
|------|------|-------|--------|
| `data/unified_training_data.csv` | ~954 | Unified MultiTask Model | `scripts/build_integrated_training_data.py` |
| `data/merged_xgb_training.csv` | ~370 | XGBoost Developability Predictor | `scripts/retrain_devpred_esm2.py` (consumer) |

---

## 1. `unified_training_data.csv`

**Purpose:** Train the Unified MultiTask Model, which predicts eight
developability properties simultaneously (Tm, aggregation risk, stability,
viscosity risk, hydrophobicity, potency, ka, nu).

**Contents:** 954 rows covering multiple molecule types (mAbs, bispecifics,
nanobodies, fusion proteins). Includes both real experimental data and
synthetic variants with realistic noise. Some rows have masked (NaN) task
columns -- this is intentional, as the multitask model uses masked training
so it can learn from partially-labeled molecules.

**Columns:**
`hc_sequence, lc_sequence, pI, MW_kDa, deam_sites, ox_sites,
acidic_residues, basic_residues, hydrophobicity_gravy, ka, nu, tm,
aggregation_risk, stability, viscosity_risk, hydrophobicity, potency,
seq_type`

**Generation script:** `scripts/build_integrated_training_data.py`

**Sources merged:**
1. Jain-137 (137 rows, VH+VL, 13 experimental targets)
2. merged_wetlab_training.csv (267 rows, VH+VL, 7 targets)
3. prophet_ab.csv (246 rows, VH+VL+full HC/LC, 14 targets)
4. garbinski2023_tm1.csv (86 rows, VH+VL, Tm1 only)
5. tresanco2023nbthermo_tm.csv (672 rows, nanobody VH only, Tm)
6. jain2024assessment_SEC.csv (43 rows, SEC % Monomer)
7. shanehsazzadeh2023unlocking_SEC.csv (13 rows, SEC % Monomer)
8. benchmark_sequences.json (12 molecules, full-length)

Deduplication is by VH sequence. Real experimental values are mapped to
the 8 unified task columns; missing values remain NaN (masked).

---

## 2. `data/merged_xgb_training.csv`

**Purpose:** Train the XGBoost developability predictor, which requires
every row to have complete labels for its target properties.

**Contents:** 370 rows with no missing labels. Only molecules that have
actual experimental measurements for all XGBoost target columns
(fab_tm, hic_rt, acsins, psr, stability_slope, titer, smac_rt) are included.

**Columns:**
`vh, vl, fab_tm, hic_rt, acsins, psr, stability_slope, titer, smac_rt, source`

**Consumer scripts:**
- `scripts/retrain_devpred_esm2.py` (ESM2-based XGBoost retraining)
- `scripts/retrain_weak_targets.py` (targeted retraining of weak tasks)

---

## Why Two Files?

The unified model uses **masked multi-task learning**: it can learn from a
molecule even if only 2 of 8 labels are known. This allows it to use all
954 rows. The XGBoost model requires complete feature vectors -- no NaN
values -- so it uses only the 370 fully-labeled rows.

They are not subsets of each other. The unified file has additional columns
(pI, MW_kDa, sequence features) and different column naming conventions.

## Data Flow

```
  Literature CSVs          benchmark_sequences.json
  (Jain-137, wetlab,       (12 reference molecules)
   PROPHET-Ab, etc.)                |
        |                          |
        v                          v
  build_integrated_training_data.py
        |
        +---> data/unified_training_data.csv  (954 rows, masked NaNs OK)
        |          |
        |          +---> scripts/train_unified_model.py --> models/unified/
        |
        +---> data/merged_xgb_training.csv    (370 rows, fully labeled)
                   |
                   +---> scripts/retrain_devpred_esm2.py --> models/developability/
                   +---> scripts/retrain_weak_targets.py --> models/developability/
```

## How to Regenerate

### Unified training data

```bash
cd /path/to/ProtePilot
python3 scripts/build_integrated_training_data.py
# Output: data/unified_training_data.csv
```

### XGBoost training data

The merged_xgb_training.csv is produced as a side-effect of the same
build script (it filters to fully-labeled rows and writes the XGBoost
subset). To regenerate both files at once:

```bash
python3 scripts/build_integrated_training_data.py
```

### Retraining models after regeneration

```bash
# Unified multitask model
python3 scripts/train_unified_model.py

# XGBoost developability model
python3 scripts/retrain_devpred_esm2.py
```
